# Visualization 结果说明与论文写作指导

本目录用于存放问题二基础预测模型和灵敏度分析的可视化代码与图片结果。可视化的作用主要有三个：

1. 展示本文模型与基线方法的预测效果差异；
2. 说明模型误差在站点、功能区和小时维度上的分布特征；
3. 支撑灵敏度分析结论，说明模型对关键参数和结构设定是否稳定。

当前可视化结果分为两部分：

| 目录 | 内容 |
| --- | --- |
| `basic_model\results` | 基础模型与基线方法的对比图，共 13 张 |
| `sensitivity_analysis\results` | 六类灵敏度分析图，每类 6 张，共 36 张 |

## 1. 使用原则

论文中不建议把所有图片都放入正文。建议优先选择能够支撑核心结论的图，其余图片可以作为备查结果或附录材料。

推荐优先进入正文的图包括：

| 优先级 | 图片 | 推荐理由 |
| --- | --- | --- |
| 高 | `basic_model\results\01_overall_metrics_combined.png` | 最直接展示本文模型与两个基线的总体预测效果 |
| 高 | `basic_model\results\03_station_mae_improvement.png` | 展示模型在不同站点上的改进或退化，能体现站点异质性 |
| 高 | `basic_model\results\05_zone_metrics_combined.png` | 支撑功能区分析，与第一问结论衔接 |
| 高 | `sensitivity_analysis\results\sensitivity_visualization_index.csv` 对应的各类 `01_scenario_metric_overview.png` | 用于说明灵敏度分析总体结论 |
| 中 | `basic_model\results\06_selected_hourly_fit_borrow.png` 和 `06_selected_hourly_fit_return.png` | 适合展示代表站点的逐小时拟合效果 |
| 中 | `basic_model\results\09_selected_stock_trajectories.png` | 适合连接第三问库存轨迹分析 |
| 中 | 灵敏度分析中的 `02_delta_vs_base_model.png` | 适合展示参数扰动相对于基础模型的变化 |
| 低 | 热力图类和风险类补充图 | 信息量大，适合作为附录或答辩备查 |

如果论文篇幅有限，可以采用“总览图 + 站点异质性图 + 功能区图 + 灵敏度分析总览图”的组合。

### 2.1 `visualization_index.csv`

这是基础模型可视化的索引文件，记录了每张图片的文件名和简短说明。

论文写作时可以把它当作图片清单使用，但正文中不需要引用该索引文件。它更适合用于检查图片是否生成完整。

### 2.2 `01_overall_metrics_combined.png`

该图展示三种方法在借出量和归还量合并指标上的总体对比。

图中通常包含以下方法：

| 方法 | 含义 |
| --- | --- |
| `base1` | 直接使用第 6 天对应小时数据预测第 7 天 |
| `base2` | 使用功能区平均模板进行预测 |
| `model` | 本文提出的收缩修正预测模型 |

图中通常展示的指标包括 MAE、RMSE、sMAPE、Hit10、全天总量误差和小时结构误差等。

这张图适合放在问题二“模型检验与基线比较”部分，作为最核心的模型效果总览图。

论文表述建议：

```text
从合并借出量与归还量的总体指标看，本文模型相较直接复制最近同类日的基线方法在多数误差指标上有所改善，说明引入功能区层和可信度收缩机制能够缓解单日休息日样本的随机波动。
```

需要注意，如果某些指标上 `base2` 表现优于本文模型，不要写成“本文模型全面优于所有基线”。更稳妥的写法是“本文模型相较直接复制基线更稳定，而功能区平均模板在部分指标上具有竞争力”。

### 2.3 `02_metrics_by_variable.png`

该图分别展示借出量和归还量的预测指标，而不是只看合并结果。

它回答的问题是：模型对借出量和归还量是否同样有效，或者是否存在对某一类变量预测更好的情况。

适合放在问题二模型检验部分，作为总体指标图的补充。如果论文篇幅有限，可以不放正文，只在文字中说明借出量和归还量分别进行了检验。

论文表述建议：

```text
进一步区分借出量与归还量后可以发现，模型在两类变量上的误差变化趋势基本一致，说明预测框架并非只对单一方向的流量有效。
```

如果图中借出量和归还量差异较明显，则应如实表述为“模型对借出量和归还量的改善幅度存在差异”。

### 2.4 `03_station_mae_improvement.png`

该图展示各站点层面，本文模型相对于基线方法的 MAE 改变量。

一般来说：

| 图中含义 | 解释 |
| --- | --- |
| 改变量小于 0 | 本文模型误差低于基线，说明该站点上模型有改善 |
| 改变量大于 0 | 本文模型误差高于基线，说明该站点上模型退化 |
| 改变量接近 0 | 本文模型与基线差异不明显 |

这张图非常适合放在“站点层面检验”或“模型适用性分析”部分。它能够说明模型不是在所有站点上表现完全一致，而是存在站点异质性。

论文表述建议：

```text
站点层面的 MAE 改变量表明，本文模型在多数站点上相较直接复制基线具有更低误差，但仍存在少数站点由于第 6 天和第 7 天行为差异较大而预测效果退化。这说明休息日单日样本预测仍存在不可避免的不确定性。
```

### 2.5 `04_station_mae_heatmap.png`

该图以热力图形式展示不同站点、不同预测方法下的 MAE。

它适合回答两个问题：

1. 哪些站点本身预测难度较高；
2. 哪些方法在不同站点上的误差分布更均衡。

这张图信息量较大，适合放在附录或答辩材料中。如果正文需要强调站点异质性，也可以选择性放入。

使用时应重点关注颜色明显偏深的站点，这些站点可能具有低流量、尖峰集中、偶发事件或周六周日行为差异较大的特征。

### 2.6 `05_zone_metrics_combined.png`

该图展示不同功能区下的合并预测指标。

它用于说明模型误差是否受到站点所处功能区影响，并且可以与第一问中的功能区分析形成衔接。

适合放在问题二“功能区差异分析”或“模型解释性分析”部分。

论文表述建议：

```text
按功能区汇总后，不同区域的预测误差存在差异，说明共享单车周末借还行为不仅受站点自身历史数据影响，也与站点所在区域的功能属性有关。因此，在模型中引入功能区层修正具有现实解释意义。
```

如果某些功能区误差明显较高，可以进一步解释为该区域周末行为不稳定、功能混合程度高或站点样本较少。

### 2.7 `06_selected_hourly_fit_borrow.png`

该图展示 6 个代表站点上，借出量的逐小时真实曲线与预测曲线。

它适合用于说明模型是否能够跟踪一天内的峰谷变化，而不仅仅是总量准确。

推荐放在问题二模型效果展示部分，作为“典型站点案例图”。如果图片较大，也可以只选借出量或归还量其中一张进入正文。

使用时应重点观察：

| 观察点 | 解释 |
| --- | --- |
| 峰值时间是否接近 | 判断模型是否捕捉到日内节奏 |
| 峰值高度是否接近 | 判断模型是否低估或高估高峰需求 |
| 低谷时段是否稳定 | 判断模型是否产生不合理波动 |
| 预测曲线是否过平滑 | 判断收缩修正是否削弱了局部变化 |

### 2.8 `06_selected_hourly_fit_return.png`

该图展示 6 个代表站点上，归还量的逐小时真实曲线与预测曲线。

作用与 `06_selected_hourly_fit_borrow.png` 类似，但关注归还行为。

论文中如果想节省篇幅，可以把借出量和归还量两张图合并描述：

```text
代表站点逐小时曲线表明，本文模型能够较好捕捉主要峰谷时段，但在个别站点的尖峰小时仍存在低估或高估，说明单日休息日样本对极端小时的预测能力有限。
```

### 2.9 `07_heatmap_actual_model_error_borrow.png`

该图是借出量的站点-小时热力图，通常包含真实值、模型预测值和误差。

它用于观察借出量误差在时间和空间上的集中位置。

适合放在附录或答辩材料中。如果正文讨论“哪些小时最容易出错”，也可以引用该图。

使用时建议关注：

| 观察点 | 解释 |
| --- | --- |
| 是否存在连续高误差小时 | 可能说明模型未捕捉某段时间的系统性变化 |
| 是否集中在少数站点 | 可能说明站点个体行为特殊 |
| 是否集中在高峰时段 | 说明峰值预测是主要误差来源 |

### 2.10 `07_heatmap_actual_model_error_return.png`

该图是归还量的站点-小时热力图，通常包含真实值、模型预测值和误差。

它的使用方式与借出量热力图一致，但关注归还行为。

论文中如果需要讨论“归还量误差是否与借出量误差具有不同时间分布”，可以同时使用借出量和归还量两张热力图。

### 2.11 `08_zone_average_hourly_curves.png`

该图展示不同功能区的平均逐小时借还曲线。

它回答的问题是：不同功能区是否具有不同的日内借还节奏。

这张图适合放在功能区分析部分，也可以作为第一问和第二问之间的衔接图。

论文表述建议：

```text
不同功能区的平均小时曲线存在一定差异，表明站点所在区域功能会影响周末共享单车借还节奏。该现象为第二问中引入功能区层模板提供了数据依据。
```

### 2.12 `09_selected_stock_trajectories.png`

该图展示代表站点中，由真实借还量和预测借还量分别驱动得到的库存轨迹。

它不是对真实库存的完全还原，而是用于比较预测误差对库存演化判断的影响。

适合放在问题二和问题三的衔接位置。如果第三问需要制定调度策略，这张图可以说明问题二预测结果能否作为库存演化输入。

论文表述建议：

```text
将预测借还量代入库存演化方程后，代表站点的库存轨迹与真实借还量驱动的轨迹整体趋势较为接近，说明该预测结果可为后续库存风险识别和调度优化提供基础输入。
```

需要避免的表述：

不要写成“该图还原了真实库存”。题目数据可能存在未记录调度和隐性需求流失，因此这里只能作为相对比较。

### 2.13 `10_risk_metrics_by_station.png`

该图展示各站点的运营风险相关指标，例如库存轨迹误差、空桩小时差异、满桩小时差异等。

```text
从运营风险指标看，不同站点的库存轨迹误差和空满桩风险差异并不完全一致，说明单纯追求借还量预测误差最小并不足以保证库存风险判断最准确。
```

## 4. 灵敏度分析通用图片说明

以下 6 张图在每个灵敏度分析子目录下都会出现。

### 4.1 `01_scenario_metric_overview.png`

该图展示同一类灵敏度分析中，不同情景在多个核心指标上的总体表现。

通常包含的指标有：

| 指标 | 含义 |
| --- | --- |
| MAE | 平均绝对误差 |
| RMSE | 均方根误差 |
| sMAPE | 相对误差 |
| Hit10 | 预测值落入真实值正负 10% 范围的比例 |
| Share L1 | 小时结构差异 |
| Stock MAE | 库存轨迹差异 |

这张图是每类灵敏度分析最适合放入论文正文的图。它能够直观展示不同参数设定对模型效果的影响。

论文中建议放在“灵敏度分析”小节的核心位置。如果只能选一张图展示某类灵敏度分析，优先选择这张。

### 4.2 `02_delta_vs_base_model.png`

该图展示不同情景相对于基础模型的指标变化。

它的关键含义是：

| 情况 | 解释 |
| --- | --- |
| MAE、RMSE、sMAPE、Share L1、Stock MAE 的变化量小于 0 | 当前情景优于基础模型 |
| MAE、RMSE、sMAPE、Share L1、Stock MAE 的变化量大于 0 | 当前情景差于基础模型 |
| Hit10 的变化量大于 0 | 当前情景命中率高于基础模型 |
| 变化量接近 0 | 当前参数扰动对模型影响较小 |

这张图适合用于强调“模型对参数扰动是否敏感”。如果大多数情景变化量不大，可以写模型较稳定；如果某一类参数变化导致指标明显波动，则说明该参数是模型关键设定。

### 4.3 `03_station_mae_heatmap.png`

该图展示不同情景下，各站点的 MAE 热力图。

它适合分析参数变化对不同站点的影响是否一致。

论文中可以用它支持以下结论：

```text
灵敏度分析不仅在总体层面进行，也进一步考察站点层面的误差分布。结果显示，部分站点对参数变化更敏感，说明周末共享单车预测存在明显站点异质性。
```

如果正文篇幅有限，该图建议作为附录或答辩备查。

### 4.4 `04_zone_mae_comparison.png`

该图展示不同情景下，各功能区或替代分区的 MAE 对比。

它适合分析参数变化是否对某些功能区影响更大。

在 `06_zone_partition` 中，这张图尤其重要，因为它直接比较不同分区方式下的功能区或分组误差。

需要注意：在 `06_zone_partition` 中，不同情景的“zone”定义不同。基础模型中的 zone 是功能区，而活动量分位和容量分位中的 zone 是机械分组。因此该图在该目录下更适合比较不同分区方案的总体表现，不宜把不同情景下的区名当作同一种空间含义解释。

### 4.5 `05_operational_risk_comparison.png`

该图展示不同情景下的库存风险相关指标，例如库存轨迹误差和空满桩风险差异。

它适合说明参数变化不仅影响借还量预测误差，也可能影响后续库存风险判断。

论文中可用于问题二与问题三的衔接：

```text
为避免预测精度指标与运营风险脱节，本文进一步比较不同参数情景下的库存轨迹误差和空满桩风险差异。该结果用于判断预测模型是否能够为后续调度优化提供稳定输入。
```

该图不一定必须放正文，但如果第三问较依赖库存轨迹分析，建议至少放一张代表性的风险对比图。

### 4.6 `06_mae_stock_tradeoff.png`

该图展示预测 MAE 与库存轨迹误差 Stock MAE 之间的权衡关系。

它回答的问题是：借还量预测误差较低的情景，是否也一定能带来更准确的库存轨迹。

这张图适合用于说明“模型评价不能只看 MAE”。如果某些情景 MAE 较低但 Stock MAE 较高，则说明该情景对后续库存分析并不一定最优。

论文表述建议：

```text
MAE 与库存轨迹误差的权衡图表明，逐小时借还量预测精度和库存演化精度并不总是完全一致。因此，本文在模型评价中同时考虑预测误差和库存风险指标。
```

## 5. 各灵敏度分析图表的论文用途

### 5.1 `01_credibility_weight`

该部分用于说明可信度权重公式中的不同组成部分是否合理。

建议重点使用：

| 图片 | 用途 |
| --- | --- |

## 7. 使用图表时需要注意的事项

第一，不要只凭一张图下结论。总体指标图说明平均效果，站点图说明异质性，功能区图说明空间差异，风险图说明对库存调度的影响。论文中应至少结合两类图进行解释。

第二，热力图适合发现问题，不一定适合直接放正文。热力图信息密度较高，如果论文版面有限，建议将其放入附录或用于答辩时解释异常站点。

第三，库存轨迹图和风险指标图是相对比较工具，不是真实库存还原。由于题目数据可能存在未记录调度和隐性需求流失，论文中应明确这些图用于比较不同预测方法对库存演化判断的影响。

第四，灵敏度分析中某个情景的单一指标最优，不代表该情景一定应替代基础模型。最终模型选择还需要考虑可解释性、稳健性、与第一问结论的一致性，以及对第三问库存调度的适用性。

第五，如果图中出现本文模型不如某一基线的情况，应如实说明。可以解释为该基线在部分站点或部分指标上具有竞争力，而本文模型的优势主要体现在稳健性、多层结构解释和对直接复制基线的改进。
