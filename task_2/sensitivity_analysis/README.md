# Sensitivity Analysis 结果说明与论文写作指导

本目录用于存放问题二基础预测模型的灵敏度分析代码与结果。灵敏度分析的目的不是重新提出一个新模型，而是检验基础模型中关键设定是否稳定、哪些参数最影响预测效果，以及这些设定是否具有论文中可解释的合理性。

运行全部灵敏度分析代码的命令如下：

```powershell
python "D:\Repository\mathematical_modeling_competition\2026-cumcm-campus-selection\task_2\sensitivity_analysis\run_all_sensitivity.py"
```

运行后，结果统一输出到：

```text
D:\Repository\mathematical_modeling_competition\2026-cumcm-campus-selection\task_2\sensitivity_analysis\results
```

当前所有灵敏度分析均围绕第 7 天的预测展开，即利用第 1 天至第 6 天已有数据，对第 7 天各站点、各小时的借出量和归还量进行预测，并与第 7 天真实记录进行比较。

## 1. results 顶层文件说明

### 1.1 `sensitivity_analysis_overview.csv`

这是所有灵敏度分析中最重要的总入口文件。它把 6 类灵敏度分析下的全部情景级结果合并到一张表中，适合用于制作总表、总览柱状图、不同灵敏度分析之间的横向比较图。

文件中一行对应一个灵敏度分析情景。主要字段如下：

| 字段 | 含义 | 论文用途 |
| --- | --- | --- |
| `analysis` | 灵敏度分析类别编号，例如 `01_credibility_weight` | 区分属于哪一类灵敏度分析 |
| `scenario_id` | 当前情景编号，例如 `base_model`、`volume_dominant` | 区分同一类分析下的不同参数设定 |
| `scenario_description` | 当前情景的文字说明 | 可转写为论文中参数设定说明 |
| `parameter_label` | 便于作图展示的参数标签 | 推荐用于图例或横轴标签 |
| `use_global_template` | 是否引入全局模板 | 用于说明模型结构是否启用全局层信息 |
| `finalization_mode` | 预测值后处理方式 | 用于说明是否进行非负、取整、容量约束等修正 |
| `mae` | 平均绝对误差 | 越小越好，是主要预测精度指标 |
| `rmse` | 均方根误差 | 越小越好，对较大误差更敏感 |
| `smape` | 对称平均绝对百分比误差 | 越小越好，用于相对误差比较 |
| `hit10` | 预测值落入真实值正负 10% 范围的比例 | 越大越好，用于衡量逐小时命中能力 |
| `total_relative_error` | 全天总量相对误差 | 越小越好，用于衡量总量预测是否准确 |
| `share_l1` | 24 小时结构占比的 L1 差异 | 越小越好，用于衡量日内形状是否接近 |
| `stock_mae` | 预测借还量驱动的库存轨迹与真实借还量驱动的库存轨迹之间的平均差异 | 越小越好，用于连接第三问的库存演化风险 |
| `risk_diff` | 空桩小时差异与满桩小时差异之和 | 越小越好，用于衡量运营风险判断是否接近 |
| `delta_vs_base_mae` | 当前情景相对于基础模型的 MAE 变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_rmse` | 当前情景相对于基础模型的 RMSE 变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_smape` | 当前情景相对于基础模型的 sMAPE 变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_total_relative_error` | 当前情景相对于基础模型的总量相对误差变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_share_l1` | 当前情景相对于基础模型的小时结构误差变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_stock_mae` | 当前情景相对于基础模型的库存轨迹误差变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_risk_diff` | 当前情景相对于基础模型的风险小时差异变化 | 小于 0 表示优于基础模型 |
| `delta_vs_base_hit10` | 当前情景相对于基础模型的 10% 命中率变化 | 大于 0 表示优于基础模型 |

论文写作建议：

优先从该文件提取每类灵敏度分析中 `mae` 最低或综合表现最稳定的情景，形成一张“灵敏度分析总览表”。如果论文篇幅有限，可以只展示 `mae`、`rmse`、`share_l1`、`stock_mae` 和 `delta_vs_base_mae`，其余指标放在文字中概括。

需要注意，灵敏度分析中某些情景的 MAE 可能略优于基础模型，但这不必然意味着应该替换基础模型。论文中应同时考虑预测精度、解释性、参数稳定性和是否符合问题一得到的站点功能分区结论。

### 1.2 `sensitivity_analysis_overview.md`

这是自动生成的文字版总览，概括了每一类灵敏度分析中 MAE 最优的情景，并给出简短使用说明。

该文件适合用于快速查看结果，不建议直接作为论文最终文字。论文中可以参考它给出的最优情景，但最终表述应结合各子目录下的 `*_conclusion.md` 和对应 CSV 指标。

当前结果显示：

| 灵敏度分析类别 | MAE 最优情景 | 主要含义 |
| --- | --- | --- |
| `01_credibility_weight` | `volume_dominant` | 可信度权重中更强调站点活动总量时，预测误差有所下降 |
| `02_weight_clip` | `narrow_clip_0_3_0_7` | 将可信度权重限制在较窄区间内，有助于减少极端权重带来的不稳定 |
| `03_zone_shrinkage` | `theta_0_0_global_only` | 单纯从 MAE 看，全局模板有较强解释力 |
| `04_global_template` | `global_only_template` | 全局日内结构在当前数据中具有较强稳定性 |
| `05_feasibility_correction` | `round_only` | 当前数据下简单取整已经表现较好，容量约束修正没有明显改变结果 |
| `06_zone_partition` | `base_model` | 原始人工功能分区相较其他替代分区更稳健 |

## 2. 各子目录通用文件说明

六个灵敏度分析子目录的文件结构完全一致。文件名前缀不同，但每类文件的作用相同。

### 2.1 `*_scenario_summary.csv`

该文件记录当前灵敏度分析下每个情景的总体综合指标，一行对应一个情景。

它是单项灵敏度分析中最适合进入论文正文的文件。推荐用它制作“不同参数取值下模型误差变化表”或“情景级指标对比图”。

主要字段与 `sensitivity_analysis_overview.csv` 基本一致，包括 `mae`、`rmse`、`smape`、`hit10`、`total_relative_error`、`share_l1`、`stock_mae`、`risk_diff` 以及相对于基础模型的变化量。

论文写作建议：

在每类灵敏度分析中，优先引用该文件中的 `mae`、`rmse` 和 `share_l1`。如果需要说明模型对第三问库存调度的影响，再补充 `stock_mae` 和 `risk_diff`。

### 2.2 `*_overall_metrics.csv`

该文件记录当前灵敏度分析下，不同情景、不同方法、不同变量在全体站点层面的预测指标。

一行通常由以下维度确定：

| 字段 | 含义 |
| --- | --- |
| `scenario_id` | 当前灵敏度分析情景 |
| `method` | 预测方法，通常包括直接复制基线、功能区模板基线和本文模型 |
| `variable` | 评价对象，`borrow` 表示借出量，`return` 表示归还量，`combined` 表示借出和归还合并后的总体指标 |
| `station_id` | 全体站点汇总时通常为 `ALL` |
| `zone` | 全体功能区汇总时通常为 `ALL` |

论文写作建议：

该文件适合用于说明“灵敏度分析不是只看合并指标，也分别考察了借出量和归还量”。如果论文篇幅有限，正文中可只展示 `combined`，附录或说明中指出借出量、归还量也分别计算。

### 2.3 `*_station_metrics.csv`

该文件记录站点层面的预测指标，一行对应“情景、方法、变量、站点”的组合。

主要用途：

| 用途 | 说明 |
| --- | --- |
| 站点异质性分析 | 比较不同站点对参数变化是否敏感 |
| 异常站点识别 | 找出某些情景下误差明显增大的站点 |
| 热力图绘制 | 可作为站点 MAE 热力图或站点排序图的数据源 |
| 代表站点分析 | 可挑选高误差、中等误差、低误差站点写案例分析 |

论文写作建议：

如果要写“模型在不同站点上的稳定性”，可以从该文件统计每个情景下站点 MAE 的均值、最大值、分位数，或者统计模型相对基线在哪些站点上改善、哪些站点上退化。

### 2.4 `*_zone_metrics.csv`

该文件记录功能区层面的预测指标，一行对应“情景、方法、变量、功能区”的组合。

主要用途：

| 用途 | 说明 |
| --- | --- |
| 功能区差异分析 | 比较文教区、商业区、居住区等区域的预测误差 |
| 分区合理性验证 | 检验功能区建模是否比随机或粗糙分区更合理 |
| 论文图表 | 可绘制功能区 MAE 对比图、功能区误差雷达图、功能区风险对比图 |

论文写作建议：

在说明功能区修正有意义时，优先引用该文件。特别是 `06_zone_partition` 目录下的 `zone_metrics` 可以用于说明“人工功能分区比单一全局分区、活动量分位分区、容量分位分区更能保留空间行为差异”。

### 2.5 `*_risk_metrics.csv`

该文件记录站点层面的库存轨迹与风险小时指标，一行对应“情景、方法、站点”的组合。

主要字段如下：

| 字段 | 含义 | 论文用途 |
| --- | --- | --- |
| `stock_mae` | 预测借还量驱动的库存轨迹与真实借还量驱动的库存轨迹之间的平均绝对差异 | 衡量预测误差对库存演化的影响 |
| `empty_hours_pred` | 根据预测值模拟得到的空桩小时数 | 用于估计空桩风险 |
| `empty_hours_actual` | 根据真实借还量模拟得到的空桩小时数 | 用作风险评估参照 |
| `full_hours_pred` | 根据预测值模拟得到的满桩小时数 | 用于估计满桩风险 |
| `full_hours_actual` | 根据真实借还量模拟得到的满桩小时数 | 用作风险评估参照 |
| `empty_hour_diff` | 空桩小时数差异 | 越小越好 |
| `full_hour_diff` | 满桩小时数差异 | 越小越好 |
| `risk_diff` | 空桩小时差异与满桩小时差异之和 | 越小越好 |

需要特别说明：

这里的库存轨迹是为了比较不同预测方法对库存风险判断的影响，采用相同初始库存进行相对模拟。它不应被解释为对第 7 天真实库存的完全还原，因为题目数据中可能存在未记录调度、隐性需求流失等因素。

论文写作建议：

该文件适合放在“模型检验”或“对第三问的衔接”中。可以写成：除了直接比较借还量预测误差，本文进一步将预测借还量代入库存演化方程，考察预测误差是否会放大为空桩或满桩风险判断偏差。

### 2.6 `*_station_weights.csv`

该文件记录各情景下每个站点、每类变量的可信度权重及其组成特征。

主要字段如下：

| 字段 | 含义 |
| --- | --- |
| `station_id` | 站点编号 |
| `variable` | 借出量或归还量 |
| `baseline_total` | 基准日对应站点的全天总量 |
| `zero_hour_ratio` | 基准日中零流量小时所占比例 |
| `peak_concentration` | 基准日峰值集中程度 |
| `credibility_weight` | 模型最终使用的站点可信度权重 |
| `method` | 对应预测方法 |

论文写作建议：

该文件适合用于解释“可信度不是主观给定，而是由站点活动强度、稀疏性和峰值集中性共同决定”。如果需要展示模型机制，可选择几个代表站点列出其 `baseline_total`、`zero_hour_ratio`、`peak_concentration` 和 `credibility_weight`。

### 2.7 `*_conclusion.md`

该文件是每类灵敏度分析的文字结论，自动整理了该类分析中表现较好的情景和主要现象。

论文写作建议：

可将其中的结论作为论文文字草稿，但不要直接机械复制。建议结合 `*_scenario_summary.csv` 中的具体数值进行改写，使结论既有定量依据，也保留必要的谨慎性。

## 3. 各灵敏度分析文件夹说明

### 3.1 `01_credibility_weight`

该文件夹检验“可信度权重参数”对模型结果的影响。

核心问题是：在站点可信度权重的构造中，究竟应该更重视站点活动总量、零流量小时比例，还是峰值集中程度。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `01_credibility_weight_scenario_summary.csv` | 不同可信度权重组合下的情景级综合指标 | 论文主表或灵敏度分析总表 |
| `01_credibility_weight_overall_metrics.csv` | 不同权重组合下，全体站点层面的借出、归还、合并指标 | 分别说明借出量和归还量的稳定性 |
| `01_credibility_weight_station_metrics.csv` | 不同权重组合下，各站点预测误差 | 分析哪些站点对可信度权重最敏感 |
| `01_credibility_weight_zone_metrics.csv` | 不同权重组合下，各功能区预测误差 | 分析可信度权重变化是否影响不同功能区 |
| `01_credibility_weight_risk_metrics.csv` | 不同权重组合下，各站点库存轨迹与风险小时差异 | 连接第三问库存风险分析 |
| `01_credibility_weight_station_weights.csv` | 不同权重组合下，各站点可信度权重及组成特征 | 解释权重机制本身 |
| `01_credibility_weight_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `base_model` | 基础模型采用的默认可信度权重 |
| `equal_weights` | 各组成特征等权 |
| `volume_dominant` | 更强调基准日活动总量 |
| `sparsity_dominant` | 更强调低零流量比例 |
| `smoothness_dominant` | 更强调低峰值集中程度 |
| `stability_dominant` | 同时强调低零流量比例和低峰值集中程度 |

论文写作指导：

当前结果中，`volume_dominant` 的 MAE 表现较好，说明站点在基准日的活动总量是判断该站点周末样本可信度的重要信息。论文中可以表述为：高活动量站点的小时曲线受随机波动影响相对较小，因此适度提高活动总量在可信度权重中的作用能够改善整体预测精度。

但不建议写成“只需要活动总量即可”。因为零流量比例和峰值集中程度仍然用于识别稀疏站点和尖峰站点，对解释模型稳健性有价值。

### 3.2 `02_weight_clip`

该文件夹检验“可信度截断区间”对模型结果的影响。

核心问题是：可信度权重是否允许过低或过高。如果权重过低，模型会过度依赖功能区或全局模板；如果权重过高，模型会过度相信第 6 天的单日样本。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `02_weight_clip_scenario_summary.csv` | 不同可信度截断区间下的情景级综合指标 | 论文主表或截断区间敏感性图 |
| `02_weight_clip_overall_metrics.csv` | 不同截断区间下，全体站点层面的预测指标 | 判断整体误差随截断区间变化的方向 |
| `02_weight_clip_station_metrics.csv` | 不同截断区间下，各站点预测误差 | 识别受截断区间影响较大的站点 |
| `02_weight_clip_zone_metrics.csv` | 不同截断区间下，各功能区预测误差 | 判断不同功能区是否偏好不同权重强度 |
| `02_weight_clip_risk_metrics.csv` | 不同截断区间下，库存轨迹与风险小时差异 | 判断截断区间是否影响运营风险估计 |
| `02_weight_clip_station_weights.csv` | 不同截断区间下，各站点最终可信度权重 | 说明截断机制如何改变权重分布 |
| `02_weight_clip_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `base_model` | 基础模型采用的默认截断区间 |
| `wide_clip_0_2_0_9` | 允许权重在较宽区间内变化 |
| `upper_0_9` | 提高可信度权重上限 |
| `narrow_clip_0_3_0_7` | 将权重限制在较窄区间内 |
| `conservative_0_4_0_8` | 整体偏保守的权重区间 |
| `high_trust_0_5_0_9` | 整体更相信第 6 天样本 |

论文写作指导：

当前结果中，`narrow_clip_0_3_0_7` 的误差较低，说明完全放任可信度权重在大范围内变化并不一定有利。论文中可以解释为：由于休息日训练样本只有一天，第 6 天样本既有参考价值，也存在偶然波动，因此采用适度截断能够避免模型对单日样本过度信任或过度否定。

### 3.3 `03_zone_shrinkage`

该文件夹检验“功能分区收缩强度”对模型结果的影响。

核心问题是：站点预测应当在多大程度上向功能区模板或全局模板收缩。该分析主要改变收缩参数，使模型从完全全局模板逐渐过渡到更强调功能区模板。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `03_zone_shrinkage_scenario_summary.csv` | 不同收缩强度下的情景级综合指标 | 论文中展示收缩参数稳定性 |
| `03_zone_shrinkage_overall_metrics.csv` | 不同收缩强度下，全体站点层面的预测指标 | 分析总体误差随收缩强度变化 |
| `03_zone_shrinkage_station_metrics.csv` | 不同收缩强度下，各站点预测误差 | 找出偏好强收缩或弱收缩的站点 |
| `03_zone_shrinkage_zone_metrics.csv` | 不同收缩强度下，各功能区预测误差 | 分析功能区层修正的适用范围 |
| `03_zone_shrinkage_risk_metrics.csv` | 不同收缩强度下，库存轨迹与风险小时差异 | 分析收缩强度对库存风险估计的影响 |
| `03_zone_shrinkage_station_weights.csv` | 不同收缩强度下，各站点可信度权重 | 辅助解释站点层收缩机制 |
| `03_zone_shrinkage_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `theta_0_0_global_only` | 只使用全局模板，不使用功能区差异 |
| `theta_0_5_balanced` | 功能区模板和全局模板较均衡 |
| `theta_0_6` | 功能区模板略强 |
| `base_model` | 基础模型采用的默认收缩强度 |
| `theta_0_8` | 更强调功能区模板 |
| `theta_1_0_zone_only` | 只使用功能区模板，不使用全局模板 |

论文写作指导：

当前结果中，全局模板情景在 MAE 上表现较好，说明第 7 天整体日内形状具有较强的城市级共性。但这不等于功能区划分没有意义。功能区层修正的价值更多体现在解释空间异质性、保留问题一得到的功能区差异，以及增强模型结构的可解释性。

论文中建议采用谨慎表述：从纯预测误差看，全局模板具有较强竞争力；从模型解释和空间结构一致性看，功能区收缩仍然是合理的补充。

### 3.4 `04_global_template`

该文件夹检验“是否引入全局模板”对模型结果的影响。

核心问题是：模型是否需要城市级全局日内结构作为补充。如果不引入全局模板，模型主要依赖站点自身和功能区层；如果引入全局模板，则能吸收所有站点共同的周末日内变化规律。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `04_global_template_scenario_summary.csv` | 不同模板结构下的情景级综合指标 | 论文中说明全局模板必要性 |
| `04_global_template_overall_metrics.csv` | 不同模板结构下，全体站点层面的预测指标 | 比较有无全局模板的整体效果 |
| `04_global_template_station_metrics.csv` | 不同模板结构下，各站点预测误差 | 分析哪些站点更依赖全局模板 |
| `04_global_template_zone_metrics.csv` | 不同模板结构下，各功能区预测误差 | 分析全局模板对不同功能区的影响 |
| `04_global_template_risk_metrics.csv` | 不同模板结构下，库存轨迹与风险小时差异 | 检验全局模板是否改善风险判断 |
| `04_global_template_station_weights.csv` | 不同模板结构下，各站点可信度权重 | 辅助解释模型结构变化 |
| `04_global_template_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `base_model` | 基础模型结构 |
| `no_global_template` | 不引入全局模板 |
| `global_only_template` | 只使用全局模板 |
| `half_zone_half_global` | 功能区模板和全局模板各占一半 |
| `zone_only_via_theta` | 主要使用功能区模板 |

论文写作指导：

该分析可以作为模型合理性的关键支撑。由于休息日训练样本很少，引入全局模板可以降低单个站点或单个功能区样本过少带来的偶然波动。若 `global_only_template` 或含全局模板的情景表现较好，可以写为：城市级日内结构是当前数据中较稳定的共性信息，将其作为收缩目标有助于提高模型稳健性。

同时要避免过度解释。不能因为全局模板误差较小就否定功能区划分，因为功能区划分还承担解释空间异质性和连接第一问结论的作用。

### 3.5 `05_feasibility_correction`

该文件夹检验“极端值修正与可行性后处理”对模型结果的影响。

核心问题是：预测值是否需要经过非负修正、整数化修正和容量约束修正。共享单车借出量、归还量本质上是非负整数，并且库存演化不应长期超过站点容量或低于零。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `05_feasibility_correction_scenario_summary.csv` | 不同后处理方式下的情景级综合指标 | 论文中说明可行性修正是否影响结果 |
| `05_feasibility_correction_overall_metrics.csv` | 不同后处理方式下，全体站点层面的预测指标 | 比较取整、非负、容量修正的误差变化 |
| `05_feasibility_correction_station_metrics.csv` | 不同后处理方式下，各站点预测误差 | 检验后处理是否只影响少数站点 |
| `05_feasibility_correction_zone_metrics.csv` | 不同后处理方式下，各功能区预测误差 | 检验不同功能区是否更容易触发修正 |
| `05_feasibility_correction_risk_metrics.csv` | 不同后处理方式下，库存轨迹与风险小时差异 | 检验后处理是否改善库存风险判断 |
| `05_feasibility_correction_station_weights.csv` | 不同后处理方式下，各站点可信度权重 | 权重通常不因后处理显著改变，主要用于保持结果结构一致 |
| `05_feasibility_correction_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `raw_continuous` | 保留连续预测值，不做非负和取整修正 |
| `nonnegative_continuous` | 只进行非负修正 |
| `round_only` | 进行非负和取整修正 |
| `base_model` | 基础模型采用的最终后处理方式 |

论文写作指导：

当前结果中，`round_only` 与基础模型表现接近，说明在当前数据下容量约束修正对结果影响较小。这可以作为模型稳健性的证据：可行性修正不会显著改变预测精度，但可以保证预测结果满足共享单车数量的基本物理约束。

建议论文中这样表述：本文保留可行性修正并非为了追求更低误差，而是为了保证预测结果能够作为后续库存调度分析的可行输入。

### 3.6 `06_zone_partition`

该文件夹检验“功能分区划分方式”对模型结果的影响。

核心问题是：手工标注或题目分析得到的功能分区是否真的有必要。如果把所有站点视作同一区域，或者只按活动量、容量进行机械分组，模型效果是否会明显变化。

包含文件：

| 文件 | 记录内容 | 推荐用途 |
| --- | --- | --- |
| `06_zone_partition_scenario_summary.csv` | 不同分区方式下的情景级综合指标 | 论文中说明功能分区设定是否稳健 |
| `06_zone_partition_overall_metrics.csv` | 不同分区方式下，全体站点层面的预测指标 | 比较不同分区方式的整体效果 |
| `06_zone_partition_station_metrics.csv` | 不同分区方式下，各站点预测误差 | 分析哪些站点受分区方式影响较大 |
| `06_zone_partition_zone_metrics.csv` | 不同分区方式下，各分区预测误差 | 注意不同情景下分区含义不同，不能简单横向解释区名 |
| `06_zone_partition_risk_metrics.csv` | 不同分区方式下，库存轨迹与风险小时差异 | 检验分区方式对运营风险估计的影响 |
| `06_zone_partition_station_weights.csv` | 不同分区方式下，各站点可信度权重 | 辅助解释分区变化下的权重分布 |
| `06_zone_partition_conclusion.md` | 该灵敏度分析的文字结论 | 论文文字参考 |

主要情景含义：

| 情景 | 含义 |
| --- | --- |
| `base_model` | 使用附件 1 中人工补充或题意分析得到的功能分区 |
| `single_global_zone` | 所有站点合并为一个全局分区 |
| `activity_tertile_zone` | 按基准日活动总量分为低、中、高三类 |
| `capacity_tertile_zone` | 按站点容量分为低、中、高三类 |

论文写作指导：

该分析是支持功能区划分合理性的关键证据。如果 `base_model` 在该类分析中表现最好或较稳定，可以写为：相比完全忽略功能区、按活动量机械分区或按容量机械分区，基于站点功能属性的分区更能反映共享单车借还行为的空间差异。

需要注意，`zone_metrics` 在该目录下的解释要格外谨慎。因为不同情景的“分区”定义不同，例如 `base_model` 中的分区是功能区，而 `activity_tertile_zone` 中的分区是活动量高低组。因此该文件适合用于同一情景内部比较，不适合直接把不同情景下的区名当成同一种空间含义。

## 5. 论文撰写建议

### 5.1 推荐写作结构

灵敏度分析可以放在问题二模型检验部分，建议按照以下顺序组织：

1. 先说明灵敏度分析目的：检验可信度权重、权重截断、收缩强度、全局模板、可行性修正和功能分区划分对预测结果的影响。
2. 给出一个总览表，数据来自 `sensitivity_analysis_overview.csv`，展示每类灵敏度分析的代表性最优情景及其 MAE、RMSE、小时结构误差和库存轨迹误差。
3. 重点分析可信度权重和截断区间，因为它们直接决定“第 6 天样本是否可信”的门控机制。
4. 再分析功能区收缩和全局模板，说明模型为什么同时保留站点自身、功能区层和城市全局层信息。
5. 补充可行性修正，强调它主要保证预测结果符合非负整数和库存容量约束。
6. 最后用功能区划分灵敏度分析回应第一问结论，说明功能区划分不是任意设定，而是对空间异质性的合理表达。

### 5.2 推荐进入正文的表格

建议至少整理两张表：

| 表格 | 数据来源 | 作用 |
| --- | --- | --- |
| 灵敏度分析总览表 | `sensitivity_analysis_overview.csv` | 展示六类灵敏度分析的代表情景和主要指标 |
| 功能分区划分对比表 | `06_zone_partition_scenario_summary.csv` | 支撑功能分区设定的合理性 |

如果篇幅允许，可以增加一张：

| 表格 | 数据来源 | 作用 |
| --- | --- | --- |
| 可信度参数对比表 | `01_credibility_weight_scenario_summary.csv` 和 `02_weight_clip_scenario_summary.csv` | 说明可信度权重和截断区间的稳健性 |

### 5.3 推荐进入正文的图

这些 CSV 也已经被后续可视化脚本使用。若需要从灵敏度分析结果中选图进入论文，建议优先选择：

| 图 | 数据来源 | 论文作用 |
| --- | --- | --- |
| 各情景 MAE/RMSE 对比柱状图 | `*_scenario_summary.csv` | 展示参数变化对总体预测精度的影响 |
| 相对基础模型改变量图 | `*_scenario_summary.csv` 中的 `delta_vs_base_*` 字段 | 直观看出哪些情景优于基础模型 |
| 站点 MAE 热力图 | `*_station_metrics.csv` | 展示模型在不同站点上的异质性 |
| 功能区 MAE 对比图 | `*_zone_metrics.csv` | 展示功能区差异 |
| 库存风险对比图 | `*_risk_metrics.csv` | 连接第三问运营风险 |

### 5.4 推荐表述方式

可以采用以下思路写入论文：

```text
为检验模型参数设定的稳健性，本文分别从可信度权重、可信度截断区间、功能区收缩强度、全局模板引入、极端值修正以及功能分区划分六个方面进行灵敏度分析。结果表明，模型对多数参数扰动保持相对稳定；适度强调站点活动总量、限制可信度权重的极端变化，以及保留城市级全局模板，均有助于降低预测误差。同时，基于站点功能属性的分区相较单一全局分区和机械分位分区具有更好的综合表现，说明问题一得到的功能区信息对问题二预测具有一定解释价值。
```

关于全局模板和功能区模板的关系，建议写得谨慎：

```text
从纯预测精度看，全局模板在部分指标上具有较强竞争力，说明第 7 天的共享单车日内变化存在明显城市级共性。但功能区模板能够保留不同空间功能区的行为差异，并与第一问的站点分类结果保持一致。因此，本文最终采用站点自身样本、功能区层模板和全局模板相结合的收缩预测框架，而不是完全依赖某一单一层次。
```

关于可行性修正，建议这样写：

```text
极端值修正灵敏度分析表明，非负、取整及容量约束处理对当前预测误差影响较小，但它们能够保证预测结果满足共享单车借还量的基本物理含义，并为后续库存演化和调度优化提供可行输入。因此，本文保留该后处理步骤。
```

### 5.5 需要避免的表述

不要写成“灵敏度分析证明基础模型一定最优”。灵敏度分析只能说明模型对参数变化是否稳定，不能证明某个参数组合在所有未来情形下最优。

不要写成“全局模板表现好，所以功能区划分没有意义”。全局模板反映城市级共性，功能区划分反映空间异质性，二者作用不同。

不要把 `risk_metrics` 中的库存轨迹解释为真实库存的精确还原。它是基于相同初始库存和预测借还量构造的相对比较指标，主要用于比较不同预测方法对空桩、满桩风险判断的影响。

不要只看 MAE。对于问题二，MAE 反映逐小时数值误差，`share_l1` 反映日内形状误差，`total_relative_error` 反映全天总量误差，`stock_mae` 和 `risk_diff` 反映对后续库存分析的影响。论文中应至少选择两到三类指标共同支撑结论。
